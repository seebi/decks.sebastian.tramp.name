Background

Types of Social Networks I (by Kaplan / Haenel 2010, White 2014)

  • Social Connections
    • Facebook, Google+, …
    • keep in touch with friends and family
    • more or less private content
  • Professional Networks
    • LinkedIn, Xing, …
    • used for recruiting and finding business partners
    • business related personal content (such as CV data)
  • Multimedia Sharing
    • Vimeo, Picasa, …
    • users more or less want to share content
    • public and private content

Types of Social Networks II

  • Informational Networks
    • StackOverflow, wer-weiss-was.de, …
    • content is question or topic based
    • users search for answers or want to get credits
  • Hobbies based Social Networks
    • YardShare, DigTheDirt, …
    • connect people with a specific hobby or around as specific topic
  • Academic and Educational Networks
    • Research Gate, The Student Room, …
    • users search for collaboration partners

Semantic Web and Linked Data

  • The web is distributed by design.
  • The Semantic Web is an extension to the current Web.
  • The meaning of web data is expressed in the same data space via shared ontologies.
  • Focus information over protocols

Main Research Question

To what extent can Semantic Web technologies be deployed to support the structure, the maintenance as well as the usage of Distributed Social Networks on the World Wide Web?

Approach

Based on the given motivational points and preliminaries, the following contributions were made:

  • An architecture of a Distributed Semantic Social Network
  • An implementation of the architectures protocols
  • Evaluation of the architecture based on
    • different working application prototypes
    • a social network simulation framework

  • Linked Data / WebID Authentication

Architecture: Basic Principles

  • Linked Data / WebID Authentication
    • main protocol for data publishing, retrieval and integration
    • dereferencable and interlinked resources
    • fosters extensibility and a distributive overall character
  • Decouple services and applications from the users data
    • settles ownership to the user
    • fosters privacy and pushes distribution
  • Protocol Minimalism
    • communicate RDF triples between DSSN nodes
    • do not enforce workflow or exact interpretation
    • ensures the extensibility of the data model
