Semantic Web Software

Working Definition

Semantic Web Software is Software which is used for
  • Storage, Querying and Deployment
  • Extraction, Enrichment, Editing and Collaboration,
  • Reasoning and Rule Execution,
of RDF knowledge bases.

Linked Data Lifecycle

Storage

A triplestore is a purpose-built database for the storage and retrieval of Resource Description Framework (RDF) metadata.

wikipedia:Triplestore via Jack Rusher (2003)

A quadstore can handle multiple knowledge bases / named graphs.

RDF Import / Parser

RDF can be serialized in different syntax. Widely in use are:

  • RDF/XML, 1999 (2004, revised W3C recommendation)
  • N-Triples, 2001 (internal W3C working draft)
  • Turtle, 2011 (W3C member submission)

But there are many more: N3, (2005, spec TBL), TriX (2004, by HP), GRDDL (2007), RDF/JSON (2008 by Talis), RDFa (2008)

RDF Serializations

Syntax: Turtle / N3

Three simple triples …

@prefix foaf: <http://xmlns.com/foaf/0.1/> .
@prefix rdfs: <http://www.w3.org/2000/01/rdf-schema#> .

<http://sebastian.tramp.name> a foaf:Person;
    foaf:workInfoHomepage <http://aksw.org/SebastianTramp>;
    rdfs:label "Sebastian Tramp".

Syntax: RDF / XML

… can look …

<?xml version="1.0" encoding="utf-8"?>
<rdf:RDF
   xmlns:foaf="http://xmlns.com/foaf/0.1/"
   xmlns:rdf="http://www.w3.org/1999/02/22-rdf-syntax-ns#"
   xmlns:rdfs="http://www.w3.org/2000/01/rdf-schema#">
  <foaf:Person rdf:about="http://sebastian.tramp.name">
    <rdfs:label>Sebastian Tramp</rdfs:label>
    <foaf:workInfoHomepage rdf:resource="http://aksw.org/SebastianTramp"/>
  </foaf:Person>
</rdf:RDF>

Querying

SPARQL Protocol and RDF Query Language

  • SPARQL 1.0 from January 2008
    • allows for a query to consist of triple patterns, conjunctions, disjunctions, and optional patterns (+ filter).
  • SPARQL 1.1 still a working draft (January 2012)
    • consists of specs for updating, the protocol, adds different query features and entailment regimes (reasoning, e.g. for RDF Schema)

SPARQL Example

Give me all object properties which are in use on any resource of type person:

SELECT DISTINCT ?p ?label
WHERE
{
    ?s a foaf:Person.
    ?s ?p ?o.
    ?p a owl:ObjectProperty.
    OPTIONAL {
        ?p rdfs:label ?label.
    }
}

Linked Data: The Four Rules

  1. Use URIs as names for things
  2. Use HTTP URIs so that people can look up those names.
  3. When someone looks up a URI, provide useful information, using the standards (RDF*, SPARQL)
  4. Include links to other URIs. so that they can discover more things.
http://www.w3.org/DesignIssues/LinkedData.html

Enrichment

Improve the quality of a knowledge base by adding additional statements:

Link Generation

Inter-linking knowledge bases by finding and deploying relevant links and defining link specifications.

Supervised Machine Learning

Derive axioms about a class by using its instances as positive examples.

Editing

Editing Schema / Ontology vs. editing instances.

  • For ontology editors: Give them a maximum DL power to express the domain they want.
    • Integrate reasoning and alignment services
  • Instance editors: Hide as much as possible RDF internals from the users and allow collaboration
    • Integrate linking and other enrichment services
  • For both: Do not show them any URI.

Collaboration

Any collaboration environment can be adapted for the usage in an RDF authoring application but wikis are very successful and were adapted first:

  • Text wikis with special RDF syntax (e.g. Semantic Media Wiki)
  • Data wikis as a next step (e.g. OntoWiki and KiWi)
  • User generated content often also part of the RDF knowledge base (e.g. as SIOC resources)

Reasoning

Infer logical consequences from a set of asserted facts.

Two options:

  • Materialize inferred statements in advance and in addition to the given statements
  • Query rewriting and other just-in-time methods

Full DL reasoning does not web scale.

Rule Execution

Custom rule execution is a powerful tool for knowledge workers.

  • Create rules which are not part of any entailment regime
  • Customize them for your specific needs and throw away everything you do not need
