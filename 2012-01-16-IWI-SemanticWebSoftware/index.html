Semantic Web Software

Working Definition

Semantic Web Software is Software which is used for
  • Storage, Querying and Deployment
  • Extraction, Enrichment, Editing and Collaboration,
  • Reasoning and Rule Execution,
of RDF knowledge bases.

Linked Data Lifecycle

Storage

A triplestore is a purpose-built database for the storage and retrieval of Resource Description Framework (RDF) metadata.

wikipedia:Triplestore via Jack Rusher (2003)

A quadstore can handle multiple knowledge bases / named graphs.

RDF Import / Parser

RDF can be serialized in different syntax. Widely in use are:

  • RDF/XML, 1999 (2004, revised W3C recommendation)
  • N-Triples, 2001 (internal W3C working draft)
  • Turtle, 2011 (W3C member submission)

But there are many more: N3, (2005, spec TBL), TriX (2004, by HP), GRDDL (2007), RDF/JSON (2008 by Talis), RDFa (2008)

RDF Serializations

Syntax: Turtle / N3

Three simple triples …

@prefix foaf: <http://xmlns.com/foaf/0.1/> .
@prefix rdfs: <http://www.w3.org/2000/01/rdf-schema#> .

<http://sebastian.tramp.name> a foaf:Person;
    foaf:workInfoHomepage <http://aksw.org/SebastianTramp>;
    rdfs:label "Sebastian Tramp".

Syntax: RDF / XML

… can look …

<?xml version="1.0" encoding="utf-8"?>
<rdf:RDF
   xmlns:foaf="http://xmlns.com/foaf/0.1/"
   xmlns:rdf="http://www.w3.org/1999/02/22-rdf-syntax-ns#"
   xmlns:rdfs="http://www.w3.org/2000/01/rdf-schema#">
  <foaf:Person rdf:about="http://sebastian.tramp.name">
    <rdfs:label>Sebastian Tramp</rdfs:label>
    <foaf:workInfoHomepage rdf:resource="http://aksw.org/SebastianTramp"/>
  </foaf:Person>
</rdf:RDF>

Querying

SPARQL Protocol and RDF Query Language

  • SPARQL 1.0 from January 2008
    • allows for a query to consist of triple patterns, conjunctions, disjunctions, and optional patterns (+ filter).
  • SPARQL 1.1 still a working draft (January 2012)
    • consists of specs for updating, the protocol, adds different query features and entailment regimes (reasoning, e.g. for RDF Schema)

SPARQL Example

Give me all object properties which are in use on any resource of type person:

SELECT DISTINCT ?p ?label
WHERE
{
    ?s a foaf:Person.
    ?s ?p ?o.
    ?p a owl:ObjectProperty.
    OPTIONAL {
        ?p rdfs:label ?label.
    }
}
